# Meus Projetos

## 2023-2

O cliente na 1°Api foi o professor Lucas Nadalete que tinha o problema de que gestão acadêmica de uma escola enfrenta desafios no controle de turmas, alunos e avaliações. Sem um sistema automatizado, o acompanhamento do desempenho dos alunos se torna trabalhoso e suscetível a erros, dificultando a organização das informações. Além disso, o cálculo das médias ponderadas exige precisão, e a falta de um formato padronizado para exportação de dados pode dificultar a análise e a tomada de decisões.
Para anteder a essa demanda a solução foi um sistema em python com uma tela de menu que permite o usuário se direcionar para o cadastro e gerenciamento de turmas, alunos, grupos de alunos e ciclos de atividades avaliativas. A solução automatiza o cálculo das médias ponderadas finais dos alunos com base nas notas de cada ciclo, garantindo precisão e eficiência. Além disso, os dados podem ser consultados via API em formato JSON e exportados para arquivos CSV, facilitando a organização e o compartilhamento das informações. Nesse sistema seria possível cadastrar novas turmas, alunos, grupos de alunos e ciclos de atividades avaliativas. Além disso, a proposta é fazer o cálculo das médias ponderadas finais dos alunos a partir da nota de cada ciclo e devolver todos os dados em json e exportar em arquivos csv.

## 2024-2

[Link do Repositório](https://github.com/c137santos/FATEC-API-3-SEMESTRE)

A empresa parceira desse projeto foi a GSW Soluções Integradas, instituição na qual é especializada em desenvolvimento e implantação de soluções inteligentes em Tecnologia da Informação para a gestão empresarial, tributária operações em comércio exterior e CRM. O problema da empresa é a necessidade de captura e armazenamento de notícias etratégicas, empresas que dependem de informações estratégicas precisam acompanhar constantemente notícias e dados relevantes de fontes diversas, como portais de notícias e APIs públicas. No entanto, realizar essa captura manualmente é um processo demorado, sujeito a erros e com alto risco de perda de informações valiosas.
A solução desse problema foi criar um sistema web para mapear portais de notícias estratégicas e realizar a captura automatizada e contínua de dados para formar um histórico. O sistema deve suportar tanto o registro de portais de notícias quanto de APIs de dados estratégicos, como previsões do tempo. A solução incluirá funcionalidades de cadastro de portais, APIs, tags e jornalistas, além de um processo de web scraping para coleta e armazenamento dos dados. Também deverá oferecer interfaces de consulta com filtros avançados e ser capaz de lidar com um grande volume de informações.
